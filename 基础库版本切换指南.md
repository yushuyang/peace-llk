# 微信开发者工具 - 基础库版本切换指南

## 问题原因

当前使用的基础库版本 2.30.4 存在 bug，导致 `this._getData is not a function` 错误。

## 解决方案

### 步骤1：打开详情面板

在微信开发者工具中：
1. 点击右上角的 **「详情」** 按钮
2. 在弹出的面板中找到 **「本地设置」**

### 步骤2：切换基础库版本

1. 找到 **「调试基础库」** 下拉菜单
2. 从 **2.30.4** 改为 **2.19.4** 或 **2.14.4**
3. 等待重新编译完成

### 推荐的基础库版本

- ✅ **2.19.4** - 稳定版本，推荐
- ✅ **2.14.4** - 较老但非常稳定
- ❌ **2.30.4** - 有 bug，避免使用
- ❌ **3.14.3** - 灰度版本，可能不稳定

### 步骤3：清除缓存并重新编译

1. 点击菜单 **「清缓存」** → **「全部清除」**
2. 点击 **「编译」** 按钮

## 如果切换后仍有问题

### 方案A：删除 project.config.json 中的 libVersion

```bash
cd ~/peace-llk
# 删除 libVersion 字段，让开发者工具自动选择合适的版本
```

### 方案B：完全重新导入

1. 在微信开发者工具左侧，右键项目 → **「删除项目」**
2. 完全退出开发者工具 (Cmd+Q)
3. 重新打开开发者工具
4. 导入项目 ~/peace-llk/
5. 确保选择 **2.19.4** 或 **2.14.4** 基础库

## 验证修复

切换基础库后，Console 应该不再显示：
```
[渲染层错误] Uncaught TypeError: this._getData is not a function
```

## 注意事项

- 基础库版本越低，兼容性越好但功能越少
- 基础库版本越高，功能越多但可能有 bug
- 开发阶段建议使用稳定版本（2.19.x）
